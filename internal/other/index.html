    其他 Internal 接口¶

    资产查询、交易同步、预约开卡等。

    交易记录查询¶

    供内部服务查询用户交易流水。

    请求

    POST /internal/transaction/query

    参数

    参数 类型 必填 说明
    uid long — 用户 UID
    brand string ✅ 品牌（仅支持 BYD）
    types list — 交易类型筛选
    statusList list — 状态筛选
    transactionCurrency string — 原始交易币种
    currency string — 结算币种
    startTime long ✅ 开始时间戳（毫秒）
    endTime long ✅ 结束时间戳（毫秒）

    响应

    { "code": 0, "msg": "success", "data": [ ...CardTransaction ] }

    查询用户卡片¶

    请求

    GET /internal/query_card?uid={uid}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    brand string ✅ 品牌

    响应

    { "code": 0, "msg": "success", "data": [ ...Card ] }

    查询 CardUser¶

    查询用户的卡片风控配置。

    请求

    GET /internal/card_user?uid={uid}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    brand string ✅ 品牌

    响应

    { "code": 0, "msg": "success", "data": { ...CardUser } }

    申请数量统计¶

    统计指定品牌下虚拟卡申请数量。

    请求

    GET /internal/apply_count?status={status}&brand={brand}
    参数 类型 必填 说明
    status string — 审核状态筛选
    brand string ✅ 品牌

    响应

    { "code": 0, "msg": "success", "data": 123 }

    预约开卡¶

    请求

    POST /internal/reserve?uid={uid}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    brand string ✅ 品牌

    预约状态¶

    请求

    GET /internal/reserve_record?uid={uid}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    brand string ✅ 品牌

    响应

    { "code": 0, "msg": "success", "data": { "reserved": true } }

    预约分页¶

    请求

    POST /internal/card_reserve_page
Content-Type: application/json

    请求体（CardReserveQuery）

    参数 类型 必填 说明
    brand string ✅ 品牌
    page int — 页码，默认 1
    rows int — 每页条数，默认 10

    响应

    { "code": 0, "msg": "success", "data": { ...PageInfo } }

    审核失败原因¶

    查询用户最新申请被拒绝/失败的原因。

    请求

    GET /internal/query_card_reject_reason?uid={uid}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    brand string ✅ 品牌

    响应

    { "code": 0, "msg": "success", "data": "提供的身份文件无效" }

    说明

    • 仅在最新申请状态为 REJECT 或 FAIL 时返回原因
    • 其他状态返回 null

    用户卡片总余额¶

    汇总用户在某品牌下全部有效卡片的余额。

    请求

    GET /internal/card_user_balance?uid={uid}&brand={brand}

    | 参数 | 类型 | 必填 | 说明 | |---|:|---:|:---:|---| | uid | long | ✅ | 用户 UID | | brand | string | ✅ | 品牌 |

    响应

    {
  "code": 0,
  "msg": "success",
  "data": {
    "uid": 10001,
    "brand": "BYD",
    "totalBalance": 1234.56,
    "cardCount": 2,
    "currency": "USD"
  }
}

    UserTotalBalanceVO 字段说明

    : uid : long — 用户 UID

    : brand : string — 品牌

    : totalBalance : decimal — 全部有效卡片余额之和（已删除的卡不计入）

    : cardCount : int — 有效卡片数量

    : currency : string — 币种（USD）

    报告补发¶

    POST /internal/report_resend?date={date}

    | 参数 | 类型 | 必填 | 说明 | |---|:|:|:---:|---| | date | string | ✅ | 日期，格式 YYYY-MM-DD |

    仅运维/开发

    该接口仅用于开发/运维测试，生产环境不可用。
