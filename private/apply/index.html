    申请流程 — /private/apply¶

    申请全流程包含 12 个接口，覆盖用户从"是否有资格申请"到"查看审核结果"的全链路。

    提交申请¶

    用户提交一张新卡片申请。

    请求

    POST /private/apply/submit
Content-Type: application/json

    请求体（ApplyForm）

    参数 类型 必填 说明
    lastname string ✅ 姓（英文/数字，最多 20 字符，姓+名总和 < 27）
    firstname string ✅ 名（英文/数字，最多 20 字符）
    phoneCode string ✅ 手机区号，如 852
    brithDay string ✅ 生日，格式 YYYY-MM-DD
    idDocumentType int ✅ 证件类型（见下表）
    idDocumentNumber string ✅ 证件号码
    phone string ✅ 手机号
    email string ✅ 邮箱
    country string ✅ 国家，如 USA
    city string ✅ 城市（最多 40 字符）
    postcode string ✅ 邮政编码
    phoneCountryCode string — 手机国家二字码，如 US（可不传，自动推导）
    province string — 州/省
    address string — 街道地址（最多 255 字符）
    street string — 街道（最多 255 字符）
    countryForDelivery string — 收货国家（实体卡必填）
    cityForDelivery string — 收货城市
    addressForDelivery string — 收货地址
    streetForDelivery string — 收货街道
    postcodeForDelivery string — 收货邮编
    virtual boolean — 是否虚拟卡，默认 true
    channel string — 渠道编码（默认 eusd）

    证件类型（idDocumentType）

    值 类型
    1 护照（Passport）
    2 健康卡（Health Card）
    3 身份证（National ID）
    4 税务 ID（Tax ID）
    5 社保卡（Social Service）
    6 驾照（Driver Licence）

    响应示例

    {
  "code": 0,
  "msg": "success",
  "data": 12345
}

    返回值为申请记录 ID（long）。

    防抖

    提交后 300 秒内不可重复提交。

    国家支持检查¶

    检查当前用户所在国家/地区是否支持开卡申请。

    请求

    GET /private/apply/region?country={country}&phoneCode={phoneCode}

    参数

    参数 类型 必填 说明
    country string ✅ 国家代码，如 USA
    phoneCode string — 手机区号

    响应

    { "code": 0, "msg": "success", "data": true }

    data: true = 支持开卡；false = 不支持。

    支持申请国家列表¶

    获取所有支持卡片申请的国家及其限制。

    请求

    GET /private/apply/region_list

    响应示例

    {
  "code": 0,
  "msg": "success",
  "data": [
    {
      "countryEn": "United States",
      "countryCode": "USA",
      "countryCode2": "US",
      "phoneCode": "1",
      "physicalCard": false,
      "virtualCard": false
    }
  ]
}

    字段说明

    | 字段 | 类型 | 说明 | |---|---| | countryEn | string | 国家英文名 | | countryCode | string | 国家三字码（ISO 3166 alpha-3） | | countryCode2 | string | 国家二字码（ISO 3166 alpha-2） | | phoneCode | string | 国际区号 | | physicalCard | boolean | 是否支持实体卡 | | virtualCard | boolean | 是否支持虚拟卡 |

    手机号格式校验¶

    校验手机号是否符合指定国家的格式规范。

    请求

    GET /private/apply/phone_check?phone={phone}&phoneCode={phoneCode}

    参数

    参数 类型 必填 说明
    phone string ✅ 手机号
    phoneCode string — 区号（与 countryCode 二选一）
    countryCode string — 国家二字码（优先）

    响应

    { "code": 0, "msg": "success", "data": true }

    data: true = 格式正确；false = 格式错误。

    防抖

    300 秒内不可重复校验。

    申请详情¶

    获取当前用户最新一条申请记录的完整信息。

    请求

    GET /private/apply/detail

    响应示例

    {

  "msg": "success",
  "data": {
    "id": 123,
    "uid": 10001,
    "cardType": "silver",
    "bin": "400000",
    "firstname": "John",
    "lastname": "Doe",
    "virtual": true,
    "birthDay": "1990-01-01",
    "idDocumentType": 1,
    "phoneCode": "1",
    "phone": "xxx",
    "email": "user@example.com",
    "country": "USA",
    "state": "CA",
    "city": "SF",
    "address": "xxx",
    "postcode": "94000",
    "countryForDelivery": "USA",
    "cityForDelivery": "SF",
    "addressForDelivery": "xxx",
    "postcodeForDelivery": "94000",
    "status": "SUCCESS",
    "admin": "system",
    "remark": null,
    "updateTime": "2025-01-01T12:00:00",
    "createTime": "2025-01-01T10:00:00",
    "brand": "BYD",
    "newCardRemind": true,
    "cardImage": "https://...",
    "cardPrivateInfo": { ... }
  }
}

    申请状态（status）

    值 说明
    WAIT 等待审核
    PROCESSING 审核中
    SUCCESS 审核通过
    REJECT 审核拒绝
    FAIL 申请失败

    newCardRemind: true 表示有新卡到达提醒；cardPrivateInfo 仅在卡片已激活时返回。

    字段说明（CardApplyVO）

    : id : long — 申请记录 ID

    : uid : long — 用户 ID

    : cardType : string — 卡片等级

    : bin : string — 卡片号段

    : firstname : string — 名

    : lastname : string — 姓

    : virtual : boolean — 是否虚拟卡

    : birthDay : string — 生日

    : idDocumentType : int — 证件类型（@IdDocumentType）

    : idDocumentNumber : string — 证件号

    : phoneCode : string — 手机区号

    : phone : string — 手机号

    : email : string — 邮箱

    : country : string — 国家

    : state : string — 州或省

    : city : string — 城市

    : street : string — 街道

    : address : string — 街道地址

    : postcode : string — 邮政编码

    : countryForDelivery : string — 收货国家

    : cityForDelivery : string — 收货城市

    : addressForDelivery : string — 收货街道

    : streetForDelivery : string — 收货街道地址

    : postcodeForDelivery : string — 收货邮政编码

    : status : string — 状态（@CardApplyStatusEnum）

    : admin : string — 审核人员

    : remark : string — 备注

    : updateTime : datetime — 修改时间

    : createTime : datetime — 创建时间

    : brand : string — 品牌

    : newCardRemind : boolean — 新卡提醒

    : cardImage : string — 卡片图片

    : cardPrivateInfo : object — 卡片详细信息（仅在已激活时返回）

    卡片图片¶

    获取用户当前可展示的卡片图片 URL（用于申请页展示）。

    请求

    GET /private/apply/card_image

    响应

    { "code": 0, "msg": "success", "data": "https://..." }

    最近申请记录¶

    获取当前用户最近一条申请记录。

    请求

    GET /private/apply/recent

    响应

    同 申请详情，仅返回最近一条。

    角色校验¶

    校验当前登录用户是否有资格申请卡片（检查用户类型）。

    请求

    GET /private/apply/check_user_type

    响应

    { "code": 0, "msg": "success", "data": true }

    允许申请的用户类型

    仅 COMMON、TRADER、TRADER_SWAP 类型用户可申请卡片。

    最新申请列表¶

    查询当前用户的最新申请记录，支持按类型筛选。

    请求

    GET /private/apply/apply_latest?virtual={virtual}
    参数 类型 必填 说明
    virtual boolean — true = 仅虚拟卡，false = 仅实体卡，不传 = 返回全部类型最新各一条

    响应示例

    {
  "code": 0,
  "msg": "success",
  "data": {
    "list": [ ...CardApplyVO ],
    "total": 2
  }
}

    审核结果是否已读¶

    查询当前用户是否点击过审核结果页面。

    请求

    GET /private/apply/audit_read

    响应

    { "code": 0, "msg": "success", "data": false }
// data: false = 未读，true = 已读

    标记审核结果已读¶

    用户点击"查看审核结果"按钮后标记已读（避免重复弹窗提醒）。

    请求

    POST /private/apply/audit_mark

    响应

    { "code": 0, "msg": "success", "data": true }
