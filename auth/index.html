    认证与响应规范¶

    认证方式¶

    Public 接口¶

    无认证，所有请求均可访问。

    Private / Admin 接口¶

    使用 @LoginAuth 注解，拦截器从 HTTP Header 中解析：

    Header 说明 示例
    uid 用户 ID 12345
    brand 品牌代码 BYD、UNI

    这些 Header 由网关在用户登录后统一注入，调用方无需自行处理。

    Internal 接口¶

    除通用 Header 外，还需额外的签名校验：

    Header 说明
    nonce 时间戳字符串
    sign 签名（ApiEncoder.encrypt(body + nonce, secret)）

    签名算法（ApiEncoder）将请求 Body（不含 params 字段）和 nonce 拼接后加密，与 sign Header 比对。

    统一响应格式¶

    RestResult（通用）¶

    {
  "code": 0,
  "msg": "success",
  "data": {}
}
    字段 类型 说明
    code int 0 = 成功，非 0 = 失败
    msg string 状态描述（成功固定为 success）
    data object/null 返回数据，失败时为 null

    BaseResponse（Internal 专用）¶

    {
  "code": 0,
  "msg": "success",
  "data": {}
}

    结构同 RestResult，仅在部分 Internal 接口使用。

    常见错误码¶

    场景 code 示例 说明
    成功 0 —
    参数错误 400 请求参数校验失败
    业务异常 >0 具体含义见各接口说明
    重复提交 599 防抖命中（相同 txid）
    余额不足 600 转出时余额不足
    记录不存在 591 查询的划转记录不存在

    详细错误码含义见 错误码参考。

    频率限制¶

    注解 作用 默认间隔
    @PreventRepeat 防重复提交 300 秒
    @PreventSpeed 防快速连续请求 1 秒
    @PreventRepeat(interval=2500) reserve 等特殊接口 2500 秒

    命中频率限制时返回 code=500，msg 描述具体限制类型。
