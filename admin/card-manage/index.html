    卡片管理 — /admin/card_*¶

    管理后台对卡片、交易、划转的操作管理。

    卡片列表分页¶

    请求

    POST /admin/card_page
Content-Type: application/json

    请求体（ManageCardQuery）

    参数 类型 必填 说明
    brand string ✅ 品牌
    uid long — 用户 UID
    cardId string — 卡片 ID
    status string — 卡片状态
    page int — 页码，默认 1
    rows int — 每页条数，默认 10

    响应示例

    {
  "code": 0,
  "msg": "success",
  "data": {
    "list": [ ...ManageCardVO ],
    "total": 100
  }
}

    ManageCardVO 字段说明

    : id : long — 数据库主键 ID

    : cardId : string — 卡片 ID

    : uid : long — 用户 UID

    : userType : int — 用户类型

    : userCategory : int — 用户分类（状态）

    : username : string — 用户名

    : email : string — 邮箱

    : virtual : boolean — 是否虚拟卡

    : cardNo : string — 完整卡号（含脱敏）

    : cardName : string — 卡片姓名

    : expYear : string — 有效期年份

    : expMonth : string — 有效期月份

    : status : string — 卡片状态

    : totalRecharge : decimal — 累计充值（USD）

    : totalWithdraw : decimal — 累计提现

    : totalSpend : decimal — 累计消费

    : balance : decimal — 当前余额（USD）

    : monthlyTotal : decimal — 月限额

    : dailyTotal : decimal — 日限额

    : perTransaction : decimal — 单笔限额

    : velocityModType : string — 限额修改类型

    : allowUserModifyLimit : boolean — 是否允许用户自行修改限额

    : createTime : datetime — 开卡时间

    : deleteTime : datetime — 销卡时间

    : cardUserRole : string — 持卡人角色（CardUserRoleEnum）

    : remark : string — 备注

    : brand : string — 品牌

    交易记录分页¶

    请求

    POST /admin/card_transaction_page
Content-Type: application/json

    请求体（CardTransactionQueryForm）

    参数 类型 必填 说明
    brand string ✅ 品牌
    uid long — 用户 UID
    id long — 交易 ID
    type string — 交易类型
    status string — 状态
    transactionTimeGe datetime — 交易时间 >=（默认 365 天前）
    transactionTimeLe datetime — 交易时间 <=（默认当前）
    page int — 页码，默认 1
    rows int — 每页条数，默认 10

    响应示例

    {
  "code": 0,
  "msg": "success",
  "data": {
    "list": [ ...CardTransactionAdminVO ],
    "total": 100
  }
}

    CardTransactionAdminVO 在 CardTransactionVO 基础上增加：

    | 字段 | 类型 | 说明 | |---|---| | email | string | 持卡人邮箱 | | cardUserRole | string | 持卡人角色 | | statusRemark | string | 状态备注（根据类型/状态/退款情况动态生成） |

    交易汇总报表¶

    按时间段汇总主要交易数据。

    请求

    POST /admin/main_transaction
Content-Type: application/json

    请求体（MainTransactionExportForm）

    参数 类型 必填 说明
    transactionTimeGe datetime — 开始时间（默认 30 天前）
    transactionTimeLe datetime — 结束时间（默认当前）

    响应示例

    {
  "code": 0,
  "msg": "success",
  "data": [
    {
      "date": "2025-01",
      "totalConsume": 100000,
      "totalRefund": 5000,
      "totalFee": 1000
    }
  ]
}

    划转记录分页¶

    请求

    POST /admin/card_transfer_page
Content-Type: application/json

    请求体（CardTransferQueryForm）

    参数 类型 必填 说明
    brand string ✅ 品牌
    uid long — 用户 UID
    type string — 划转类型
    page int — 页码，默认 1
    rows int — 每页条数，默认 10

    CardTransferVO 字段说明

    : id : long — 划转记录 ID

    : txid : string — 事务 ID

    : type : string — 划转类型：IN（转入）/ OUT（转出）

    : status : string — 状态：WAIT / SUCCESS / FAILURE / REFUND

    : currency : string — 币种

    : spotAmount : decimal — 现货钱包金额（正向 = 充值）

    : cardAmount : decimal — 卡片到账金额（含手续费）

    : fee : decimal — 手续费

    : createTime : datetime — 创建时间

    : cardNo : string — 卡号（含脱敏）

    操作日志分页¶

    请求

    POST /admin/card_action_page
Content-Type: application/json

    请求体（CardActionQuery）

    参数 类型 必填 说明
    brand string ✅ 品牌
    cardId string — 卡片 ID
    uid long — 用户 UID
    page int — 页码，默认 1
    rows int — 每页条数，默认 10

    操作人格式化规则

    operator 值 显示为
    等于 uid 用户
    webhook 第三方
    其他 客服：{operator}

    划转开关¶

    开启或关闭用户的划转功能（风控操作）。

    请求

    POST /admin/card_switch
Content-Type: application/json

    请求体（CardUserSwitchForm）

    参数 类型 必填 说明
    uid long ✅ 用户 UID
    state int ✅ 1 = 开启划转，0 = 关闭划转
    remark string — 关闭原因
    brand string ✅ 品牌

    响应

    { "code": 0, "msg": "success", "data": null }

    冻结卡片¶

    管理员冻结或解冻指定卡片。

    请求

    POST /admin/card_frozen?uid={uid}&cardId={cardId}&frozen={frozen}&admin={admin}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    cardId string ✅ 卡片 ID
    frozen boolean ✅ true = 冻结，false = 解冻
    remark string — 备注
    admin string ✅ 操作管理员
    brand string ✅ 品牌

    停用卡片¶

    请求

    POST /admin/card_suspend?uid={uid}&cardId={cardId}&suspend={suspend}&admin={admin}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    cardId string ✅ 卡片 ID
    suspend boolean ✅ true = 停用，false = 启用
    remark string — 备注
    admin string ✅ 操作管理员
    brand string ✅ 品牌

    删除卡片¶

    请求

    POST /admin/card_delete?uid={uid}&cardId={cardId}&admin={admin}&brand={brand}
    参数 类型 必填 说明
    uid long ✅ 用户 UID
    cardId string ✅ 卡片 ID
    remark string — 备注
    admin string ✅ 操作管理员
    brand string ✅ 品牌

    设置限额¶

    管理员为指定卡片设置消费限额。

    请求

    POST /admin/card_velocity_control
Content-Type: application/json

    请求体（UserLimitForm）

    参数 类型 必填 说明
    uid long ✅ 用户 UID
    cardId string ✅ 卡片 ID
    monthlyTotal decimal ✅ 月累计限额
    dailyTotal decimal ✅ 日累计限额
    perTransaction decimal ✅ 单笔限额
    operator string ✅ 操作管理员
    remark string — 备注
    allowUserModifyLimit boolean — 是否允许用户自行修改限额
    brand string ✅ 品牌

    更新卡片信息¶

    管理员更新卡片备注或持卡人角色。

    请求

    POST /admin/card_update
Content-Type: application/json

    请求体（CardUpdateForm）

    参数 类型 必填 说明
    id long ✅ 卡片数据库主键 ID
    remark string — 备注
    cardUserRole string — 持卡人角色（见 CardUserRoleEnum）
    admin string ✅ 操作管理员
    brand string ✅ 品牌

    账户余额¶

    查询卡片账户总余额（供对账使用）。

    请求

    GET /admin/card_accountInfo?brand={brand}
    参数 类型 必填 说明
    brand string ✅ 品牌

    响应

    {
  "code": 0,
  "msg": "success",
  "data": { "balance": "1234567.89" }
}
